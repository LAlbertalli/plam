# Security & Sandboxing

## Overview
Ensures that code generated by LLMs runs safely without compromising the host system.

## Sandboxing Details
- **Ephemeral Containers**: Code execution happens via the Docker SDK in a pre-configured Docker base image. This avoids runtime delays (like `pip install`) and is instantly destroyed after the execution finishes.
- **Network Isolation**: By default, containers are spun up with `--network none`.
- **File System**: Host directories are strictly isolated and not mounted unless explicitly configured and enabled in the Web UI for a specific skill.
- **Credential Injection**: API keys are passed securely via environment variables during the `docker run` command and are never stored in the code.

## Security Review Agent & Approval
A specialized, lightweight agent that pre-screens all generated shell/python commands. If a command is flagged as risky (e.g., file deletion, unexpected network access), it halts the execution pipeline and triggers a **Human-in-the-loop approval request** in the Web UI.
